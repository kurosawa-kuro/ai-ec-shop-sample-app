# 01 要件 — Lumière Select

## 目的

コンセプトは **「AI で困りごとを解決する EC」**。若い女性向けの美容・ギフトセレクトショップという設定の営業デモ。

中心にあるのは「買い物の困りごと」の解決だ。「肌が弱い女友達に贈るギフト、どれが適切か分からない」——
このように *どの商品が自分の状況に適切なのかが分からない* という困りごとを訪問者が自然文で伝えると、AI が
36 品のカタログから **適切な 3 品 + それぞれを薦める理由 + 精度を上げる追加質問** を即時返して解決する。
気に入った候補はそのままカートに入れ、会員登録なしで注文フロー（カート → 注文者情報 → 完了 → 履歴）まで体験できる。

「商品を探してください」より心理的ハードルが低く、贈り手は選定理由つきで安心して購入に進める。
店舗側は `/admin-demo`（店舗インサイト）で相談傾向・AI 経由注文・売れ筋・改善提案を確認できる。

## 優先順位

```
AI コンシェルジュの動作（concierge Edge Function + 相談 UI + 3件提案＋理由）
↓ 商品一覧の AI リランキング（recommend-products）
↓ 購入フロー（カート → 注文者情報 → 完了 → 注文履歴）
↓ AI 障害時フォールバック
↓ 店舗インサイト（/admin-demo）のデモ表現
```

## 範囲

**対象:**

- トップの相談入力 → コンシェルジュへの遷移
- AI コンシェルジュ（3 件提案・各理由・追加質問・カート追加導線）
- 商品一覧（36 品）＋カテゴリ / フリーワード絞り込み＋ AI おすすめ順リランキング
- 商品詳細、カート、checkout（デモ決済）、注文完了、注文履歴
- 店舗インサイト `/admin-demo`（相談件数・AI 経由注文・デモ売上・売れ筋・改善提案）
- Supabase Edge Functions 経由の AI 呼び出し（API キー非露出）
- Cloudflare Pages デプロイ

**非対象（この段階では）:**

- 実決済・在庫管理・配送
- 会員登録・ログイン・認証
- 注文・相談のサーバーサイド永続化（localStorage のみ）
- 商品マスタの管理 CRUD 画面

## ユーザー

| ユーザー | 説明 |
|---|---|
| ギフト購入者 | 美容・ギフトに詳しくない贈り手。予算・相手・シーン・不安を自然文で相談する |
| 店舗 / 営業先 | `/admin-demo` で相談傾向・AI 経由注文・売れ筋・改善提案を見る（デモ閲覧） |

## ユースケース

| ID | ユースケース | 成功条件 |
|---|---|---|
| UC-001 | 相談文からギフト 3 件の提案を受ける | `summary` / `recommendedProductIds` / `reasons` が表示され、推薦カードが 3 件出る |
| UC-002 | 各商品の「渡しやすい理由」を見て選ぶ | 提案ごとに一言理由が表示される |
| UC-003 | 追加質問に答えて精度を上げる | `followUpQuestion` が表示され、再送信できる |
| UC-004 | 提案からそのままカートに追加する | `source=concierge` でカートに入り、AI 経由注文として記録される |
| UC-005 | 商品一覧を AI でおすすめ順に並べ替える | `rankedProductIds` 順に並び、各商品に理由が付く |
| UC-006 | 購入フローを最後まで通す | 注文番号 `DEMO-YYYYMMDD-NNN` が発行され、注文履歴に反映される |
| UC-007 | AI 障害時も詰まらない | フォールバック文言と定番候補（または通常順）で操作を継続できる |
| UC-008 | 店舗インサイトで実績を見る | 相談件数・AI 経由注文・デモ売上・売れ筋・改善提案が表示される |

## 入力（相談）

| フィールド | 必須 | 説明 |
|---|---|---|
| 相談文（query） | 必須 | 予算・相手・シーン・不安を自然文で。200 字上限（Edge Function 側で切り詰め） |
| 商品 ID 候補（productIds） | 任意 | 提案対象を特定商品に絞りたい場合に渡す |

## 商品スキーマ

商品カタログは `app/src/data/products.js`（36 品）。1 品の形:

```js
{
  id: 'skincare-001',
  name: 'モイストバランス ローション',
  category: 'スキンケア',
  price: 3200,
  tags: ['保湿', '敏感肌向け', '初心者向け'],
  giftFor: ['彼女', '妻', '友人'],
  description: '毎日のケアに取り入れやすい、さらっと使える保湿ローション。',
  aiReason: '軽い使い心地で好みが分かれにくく、初めてのスキンケアギフトにも選びやすいです。',
  imageUrl: '/images/products/skincare-001.jpg',
}
```

## AI レスポンス スキーマ

**コンシェルジュ（concierge）**

```json
{
  "summary": "予算内で失敗しにくいギフト候補です。",
  "recommendedProductIds": ["gift-001", "skincare-001", "relax-001"],
  "reasons": ["見栄えがよく好みが分かれにくいです", "軽い使い心地で選びやすいです", "香り控えめで渡しやすいです"],
  "followUpQuestion": "相手の方は普段スキンケアをされますか？"
}
```

**商品一覧の AI リランキング（recommend-products）**

```json
{
  "message": "彼女向けの高見えギフト順に並べ替えました",
  "rankedProductIds": ["skincare-003", "gift-001", "relax-001"],
  "reasons": { "skincare-003": "特別感のある高見えクリーム" }
}
```

## 制約

| 区分 | 制約 |
|---|---|
| セキュリティ | `DEEPSEEK_API_KEY` はブラウザに露出しない。Supabase Edge Functions 経由でのみ呼び出す |
| コスト | 相談文・クエリは 200 字上限。リランキング候補は 15 件上限。AI 利用は 1 ブラウザあたり 1 日 50 回（コンシェルジュ＋リランキング合算） |
| 耐障害性 | AI 障害時はフォールバック（定番 3 件 / 通常順）と説明文言で操作を継続できる |
| データ | 注文・相談・カート・利用回数は localStorage のみ。サーバーサイドに保存しない |
| 表現 | 効能・効果を断言しない（薬機法配慮）。「使いやすい」「失敗しにくい」「ギフト感がある」等の表現を使う |
