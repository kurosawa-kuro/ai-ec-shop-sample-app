# 困りごとテンプレート入口（P0）

## Goal

「何を書けばいいか分からない」訪問者向けに、困りごと起点の **相談チップ（テンプレート）** を用意し、押すだけで相談＝解決に入れるようにする。自由文が書けなくても AI 体験への導入を軽くする。

## FB 反映（owner・P0）

トップの自然文入力の**前に**相談チップを置く。チップ例:

```text
肌が弱い友人に贈りたい
美容に詳しくない相手に失敗しにくいもの
予算5,000円で高見えするギフト
香りが強すぎないものを選びたい
彼女への記念日ギフト
職場の女性に重すぎないプレゼント
```

動作: `チップを押す → query に自動入力 → そのまま concierge へ送信`。訪問者が最初から上手い相談文を書けるとは限らないため、これが効く。

## Context

- 種別: 改善（解決に入る前の入力ハードル低減）。コンセプト「困りごとを解決する」の入口設計。
- 現状の入口は自由文＋少数チップのみ:
  - トップ: `app/src/pages/Top.jsx` の `promptChips`（3件）＋検索。
  - コンシェルジュ: `app/src/pages/Concierge.jsx:11-17` の `chips`（5件）。
- チップは文字列を連結するだけで（`Concierge.jsx:133`）、「相手 × 悩み × 予算」を組み立てる導線にはなっていない。

## Scope

- 「相手（彼女/友人/母/自分…）」「悩み（肌が弱い/香りが苦手/何が喜ばれるか分からない…）」「予算」を選ぶと相談文が組み上がる軽い入口を用意（トップまたはコンシェルジュ）。
- 選択結果をそのまま `askConcierge` のクエリにする。

## Non-scope

- 全項目必須の重いフォーム化（自由文の手軽さを損なわない）。
- 商品スキーマ変更（`giftFor` / `tags` は既存を活用）。

## Skeleton

- カテゴリ別チップ群（相手 / 悩み / 予算）→ 選択を自然文テンプレに流し込み→ 既存の相談フローへ。

## Plan

1. 困りごと軸（相手・悩み・予算）の選択肢を `products` の `giftFor`/`tags` から無理なく導出。
2. 選択→相談文生成→送信の導線を実装。
3. 自由文入力との併存（どちらでも解決に入れる）。

## Acceptance Criteria

- 自由入力なしでも、選択だけで相談が成立し提案が出る。
- 生成される相談文が自然で、接地提案（#1）と噛み合う。

## Verification

- `cd app && npm run build && npm run lint && npx playwright test`
- 手動: 選択のみで提案まで到達できることを確認。
